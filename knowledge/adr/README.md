# Architecture Decision Records (ADR)

Documents significant architecture decisions and their rationale.

---

# Purpose

Use this directory to record:
- Architecture decisions that deviate from platform defaults
- Trade-off analyses for significant technical choices
- Security-related exemptions or deviations from `platform/security-rules.md`

---

# Format

Each ADR should include:
- **Title**: Short description of the decision
- **Context**: What prompted the decision
- **Decision**: What was decided
- **Consequences**: Trade-offs and implications

---

# Rule

ADRs inform but do not override `specs/`. If an ADR conflicts with a spec, the spec takes precedence.
